# 아바타 아이템 제작 가이드 (파츠별)

> 이 문서 하나만 보고 아이템 SVG를 만들 수 있도록 작성된 자기완결 가이드입니다.
> 사람이 직접 만들거나, **이 문서를 LLM에 그대로 붙여넣고 "○○ 아이템 만들어줘"라고 요청**해도 됩니다.
> 완성된 SVG 조각은 슈퍼 제어판 → 🎒 아이템 관리 → "커스텀 아이템 등록"에 붙여넣으면
> 즉시 상점에 출시됩니다(실시간 미리보기 제공).

---

## 1. 좌표계 (가장 중요)

모든 아이템은 **viewBox `0 0 320 400`** 안에 그립니다. 아바타 본체의 고정 좌표:

| 부위 | 좌표/크기 |
|---|---|
| 머리(원) | 중심 **(160, 120)**, 반지름 **56** → 머리 위 y=64, 턱 y=176 |
| 눈 | 왼쪽 **(140, 110)**, 오른쪽 **(180, 110)** |
| 코 | (160, 126) 부근 |
| 입 | (160, 145) 부근 |
| 몸통 | x 124~196, y 176~268 (어깨 y≈176, 허리 y≈250) |
| 손 | 왼손 **(103, 252)**, 오른손 **(217, 252)** |
| 다리 | 왼다리 중심 x≈145, 오른다리 중심 x≈175, y 268~334 |
| 발/신발 | y 334~358 (지면 y≈360) |
| 바닥 좌우(펫 위치) | 왼쪽 x≈55, 오른쪽 x≈265, 바닥 y≈360 |

**아바타가 정면을 보고 서 있는 모습** 기준입니다. "왼손"은 화면 왼쪽(보는 사람 기준)입니다.

## 2. 절대 규칙

1. **`id` 속성 금지.** 한 화면에 아바타가 수십 개 복제 렌더되므로 id가 있으면 충돌합니다.
   `<clipPath>`, `<linearGradient>` 등 id가 필요한 요소도 금지. 단색/불투명도로 표현하세요.
2. **`<script>`, `<foreignObject>`, `on*=` 이벤트, `javascript:` 금지** (등록 시 자동 거부).
3. 완성물은 `<svg>` 태그 없이 **내부 조각만** 제출합니다. 예:
   `<circle cx="160" cy="80" r="22" fill="#e74c3c"/>`
4. 크기 20KB 이하.
5. 색은 hex(`#rrggbb`)로. 외곽선을 줄 때는 `stroke-width 3~4`가 본체와 잘 어울립니다.

## 3. 슬롯별 작업 영역과 요령

### 모자 (hat)
- 영역: 머리 위쪽, 대략 x 100~220, y 30~100. 머리 윗면(y=64)에 걸치게.
- 볼캡 예시(챙은 ellipse로):
```svg
<path d="M106 92 Q108 46 160 44 Q212 46 214 92 Q160 76 106 92 Z" fill="#1d1f24" stroke="#000" stroke-width="3"/>
<ellipse cx="160" cy="91" rx="60" ry="10" fill="#101216"/>
```

### 헤어 (hair)
- 머리 원(160,120 r56)을 따라 두피를 덮는 형태. 이마 라인은 y≈90~100.
- 머리 색만 다른 변형이라면 기존 아이템 svg에서 fill 만 교체해도 충분.

### 얼굴 악세서리 (faceAcc)
- 눈(140/180, 110) 주변. 안경 예시:
```svg
<rect x="124" y="99" width="32" height="23" rx="8" fill="none" stroke="#2b2b2b" stroke-width="3"/>
<rect x="164" y="99" width="32" height="23" rx="8" fill="none" stroke="#2b2b2b" stroke-width="3"/>
<path d="M156 109 Q160 106 164 109" stroke="#2b2b2b" stroke-width="3" fill="none"/>
```
- 볼터치는 (126,140)/(194,140) 부근의 작은 원.

### 상의 (top) — ⚠️ 소매 마커 필수
- 몸통(124~196, 176~268)을 덮는 베이스 + **양 소매를 `sleeve-l`/`sleeve-r` 클래스 그룹으로 분리**해야
  걷기 애니메이션 때 팔과 함께 움직입니다. 아래 베이스를 그대로 쓰고 무늬만 더하는 걸 추천:
```svg
<path d="M124 176 Q160 164 196 176 L199 268 Q160 280 121 268 Z" fill="#5b8def"/>
<g class="sleeve-l"><path d="M129 178 Q112 196 106 220 L121 226 Q127 204 139 191 Z" fill="#5b8def"/></g>
<g class="sleeve-r"><path d="M191 178 Q208 196 214 220 L199 226 Q193 204 181 191 Z" fill="#5b8def"/></g>
<text x="160" y="216" text-anchor="middle" font-family="Arial" font-size="19" font-weight="bold" fill="#17181c">TEXT</text>
```
- 가슴 로고/텍스트 중심은 (160, 210~225).

### 하의 (bottom)
- 허리 y≈250부터. 바지 베이스(이대로 쓰고 색·디테일만 변경):
```svg
<path d="M125 250 Q160 262 195 250 L195 302 Q186 308 167 306 L166 285 Q160 282 154 285 L153 306 Q134 308 125 302 Z" fill="#7ba6c9"/>
<rect x="133" y="300" width="25" height="46" rx="7" fill="#7ba6c9"/>
<rect x="162" y="300" width="25" height="46" rx="7" fill="#7ba6c9"/>
```
- 반바지는 다리 rect 의 height 를 30 정도로 줄이면 됩니다(맨다리가 드러남).

### 신발 (shoes)
- y 334~358. 베이스:
```svg
<rect x="127" y="334" width="34" height="24" rx="11" fill="#d9534f"/>
<rect x="159" y="334" width="34" height="24" rx="11" fill="#d9534f"/>
```
- 양말을 추가하려면 그 위(y 308~338)에 폭 24 의 rect.

### 손 아이템 (handL / handR)
- 왼손 (103,252) / 오른손 (217,252) 주변에 들고 있는 모양으로.
- 왼손 영역 x 56~132, 오른손 영역 x 190~264, y 200~300 안에서.
- 기울이고 싶으면 `<g transform="rotate(-15 217 252)">…</g>` 처럼 손 좌표를 축으로 회전.

### 애완동물 (decoL / decoR)
- 바닥에 서 있는 동반자. 왼쪽 중심 x≈55, 오른쪽 중심 x≈265, **발이 y≈358** 에 닿게.
- 폭 60 내외의 작은 캐릭터(원+귀+눈 정도)면 충분히 귀엽습니다.

## 4. rigid 옵션

기본적으로 하의/신발은 걷기 때 다리를 따라 둘로 갈라져 움직입니다.
**한 덩어리로 두고 싶은 아이템**(치마, 긴 로브 등)은 등록 시 `rigid` 체크.

## 5. 가격 가이드

| 티어 | 가격 | 예 |
|---|---|---|
| 기본 변형 | 100~200P | 색만 다른 헤어, 단순 소품 |
| 일반 | 250~400P | 모자, 안경, 평범한 옷 |
| 고급 | 450~700P | 디테일 많은 옷, 손 아이템 |
| 프리미엄 | 800~1500P | 세트급 화려한 아이템, 희귀 컨셉 |

(참고: 가입 보너스 1,000P, 출석 10P/일, 글 작성 30P)

## 6. 제출 체크리스트

- [ ] viewBox 0 0 320 400 좌표계에 맞췄다 (위 표의 앵커 사용)
- [ ] `id=` 가 한 군데도 없다
- [ ] `<svg>` 래퍼 없이 내부 조각만 제출한다
- [ ] 상의라면 sleeve-l / sleeve-r 그룹이 있다
- [ ] 슬롯, 이름(한글), 가격을 정했다
- [ ] 슈퍼 제어판 미리보기에서 위치가 자연스럽다
