## 성공/실패 기록

### 🎯 결정적이었던 진단들

**1. Turbopack 스테일 청크 — 5번째 재발 (GiftAlert hydration)**
- hydration 수정 후에도 오류 재발 → diff 를 보니 **서버는 옛 코드, 클라는 새 코드**. 코드가 아니라 서버측 번들 캐시. 재시작 즉해결. lib/컴포넌트 수정 후 증상이 "반반 섞인" 모습이면 무조건 스테일 청크부터 의심.

**2. 모듈 인스턴스 분리 — 캐시 무효화가 안 통한다**
- 아이템 활성 토글이 화면에 안 반영. API 라우트의 `invalidateItemCache()`가 **페이지 번들의 별도 모듈 인스턴스** 캐시를 못 비움(dev 라우트별 번들/서버리스 공통). 관리 화면은 `fresh: true`로 TTL 우회.

**3. 루트 레이아웃은 소프트 내비게이션에서 재렌더되지 않는다**
- 헤더가 알림을 서버 렌더로만 조회 → 링크 이동으로는 영영 모달이 안 뜸. 클라 컴포넌트가 `usePathname` 변화+폴링으로 자가 조회하게 전환. (헤더 포인트 잔액도 같은 이유 — 구매/가챠/선물 후 `router.refresh()` 필수)

**4. toLocaleString hydration — 서버와 브라우저의 로케일이 다르다**
- "AM 8:23" vs "오전 8:23". SSR 되는 클라 컴포넌트에서 시간 표기는 **마운트 후에만** 렌더(프로덕션은 UTC 서버라 시간대까지 어긋남).

**5. CSS 키프레임은 기존 transform 을 덮는다**
- 말풍선이 오른쪽에 떴다 중앙 점프: `avPopIn`의 `scale()`만 있는 키프레임이 정렬용 `translateX(-50%)`를 애니메이션 동안 무효화. **키프레임에 translateX 포함**한 전용 keyframes 로 해결.

**6. z-index 는 스태킹 컨텍스트 안에서만 논다 — `isolation: isolate`**
- 마을 캐릭터 z(~420)가 모달(z 300) 위로 침범. 컨테이너에 isolation 한 줄로 내부 격리.

**7. storage.objects 는 SQL 로 못 지운다**
- 초기화 SQL이 42501(보호 트리거)로 전체 롤백. Storage API(service key) node 스크립트로 분리 처리. 트랜잭션이라 데이터 무손실.

**8. SMIL 애니메이션 = 이펙트 아이템의 길**
- `<animate>/<animateTransform>`는 id 불필요+sanitizer 통과+복제 안전. 횃불(불꽃 3겹 일렁+불티) 코드 검증 완료.

**9. "적립이 안 돼요" 제보 2건 모두 본문 100자 미만**
- 드롭·작성 적립 공통 조건. 코드 버그 아님 — DB 진단 스크립트(node, 임시파일)로 즉시 판정하는 패턴 정착. 에디터에 조건 안내 문구는 보류 제안 상태.

**10. 첫 페인트는 CSS 기본값으로 찍힌다**
- 런웨이 워커가 앞쪽에서 번쩍: 재마운트 첫 프레임이 CSS 기본 위치 → effect 가 옮기기 전 노출. **초기 인라인 스타일**을 시작 상태(안쪽·투명)로.
