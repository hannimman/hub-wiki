## 성공/실패 기록

### 🎯 결정적이었던 진단들

**1. "왜 supabase가 망분리에서 안 열리지?" → 아키텍처 전체를 바꾼 진단**
- **증상:** 망분리 PC 브라우저에서 기존 Supabase 주소가 안 열림.
- **1차 가설(빗나감):** 단순 일시적 차단/설정 문제일 것이다.
- **진단:** Netlify는 열리는데 supabase.co만 안 열림 → 방화벽이 도메인 단위로 supabase.co를 막음.
- **진짜 원인:** 망분리망은 `*.netlify.app`(+Functions)만 허용, `*.supabase.co` 직접 통신 차단.
- **해결:** 브라우저가 Supabase를 직접 부르지 않게 **전부 서버(Netlify) 경유** 구조로 확정. 인증도 Supabase Auth(supabase.co 직통) 대신 **자체 인증**으로.
- **교훈:** 제약(네트워크)을 **가장 먼저 end-to-end로 실측**하고 거기에 아키텍처를 맞춘다. "핑 테스트 먼저"가 전체를 구했다.

**2. "크레딧이 자꾸 줄어든다" → 배포 비용 진단**
- **증상:** Netlify 크레딧 195→164→135.6 계속 감소.
- **1차 가설(빗나감):** 화면 위치상 "Build with an AI agent" 전용 크레딧이라 우리 배포와 무관할 것이다 → **틀림.**
- **진단:** Usage & billing → Credit usage breakdown 확인.
- **진짜 원인:** **Production deploys 9회 = 135 credits (배포 1회당 ≈ 15 credits).** 개발 중 잦은 push가 주범.
- **해결:** **로컬 개발 전환**(`next dev`). Netlify env를 `netlify env:get`으로 `.env.local`에 주입해 로컬에서도 DB/로그인 동작. 배포는 기능 묶음 완성/망분리 확인 때만.
- **교훈:** 추측 대신 **Usage breakdown 같은 1차 출처**로 확인. 배포는 공짜가 아니다 → 로컬 우선.

**3. netlify dev 포트 타임아웃**
- **증상:** `netlify dev`가 "port 3000 timeout"으로 죽음.
- **진단:** 무거운 의존성(mermaid/highlight) 첫 컴파일이 netlify dev의 포트 대기 한도를 초과.
- **해결:** `netlify dev` 대신 **`next dev` 직접** + `.env.local` 수동 구성 + **Turbopack**(`--turbopack`)으로 컴파일 가속.
- **교훈:** 래퍼(netlify dev)가 느리면 프레임워크 dev를 직접 띄우고 env만 따로 주입.

**4. Mermaid "Syntax error"**
- **증상:** 첫 다이어그램만 폭탄 아이콘 + Lexical error.
- **진단:** 에러가 `...D[/login 으로]` 지점을 가리킴.
- **진짜 원인:** mermaid에서 **`[/`는 평행사변형 노드 문법** → `/]`로 안 닫혀 파싱 실패.
- **해결:** 노드 라벨이 `[/`로 시작하지 않게 수정. (에러는 페이지를 안 죽이고 폭탄+메시지로 graceful 표시되도록 처리해둔 게 도움.)
- **교훈:** 코드/다이어그램 샘플도 실제로 렌더해보고 줘야 한다.
