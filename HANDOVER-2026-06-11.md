## 성공/실패 기록

### 🎯 결정적이었던 진단들

**1. Turbopack 스테일 청크 — 세 번 재발한 끝에 패턴 확정**
- **증상(3종):** ①폴더 이동 500 `movePage is not a function` ②코스프레 구매 시 "존재하지 않는 아이템"(클라 상점엔 보이는데 구매 API만 실패) ③`/wiki/changes`에 호버 팝업 미적용.
- **진단:** 셋 다 코드가 아니라 **dev 서버(Turbopack)가 옛 모듈 청크를 캐시**한 것. lib에 **새 export를 추가**하거나 **모듈 로드 시 구성되는 데이터(ITEM_INDEX 등)를 바꾸면** 이미 컴파일된 라우트 청크가 안 따라옴.
- **해결/교훈:** dev 서버 재시작이면 즉시 해결. **카탈로그·lib 구조 변경 직후엔 선제적으로 재시작**(이후 운영 원칙으로 채택). 프로덕션 빌드에선 발생하지 않는 dev 전용 증상.

**2. HTML5 네이티브 드래그가 아예 동작 안 함**
- **증상:** 커서만 grab으로 바뀌고 dragstart가 영영 안 옴. 콘솔 에러도 없음.
- **시도→실패:** 행 전체 draggable → 내부 버튼/링크가 가로챔 → 손잡이(⠿)만 draggable → 그래도 불능(환경 자체가 네이티브 DnD 미지원).
- **해결:** **mousedown + window mousemove/mouseup 포인터 기반 직접 구현** + `elementFromPoint`로 드롭 대상 판정 + 마우스 따라다니는 칩. 환경 의존성 제로.
- **교훈:** 네이티브 API가 "조용히" 실패하면 추측 반복 대신 의존성 없는 방식으로 갈아탄다.

**3. 광장 문워크(거꾸로 걷는 느낌)**
- **진단:** 데모는 지면 ~20px/s에 걸음주기 0.76s인데 우리는 34~60px/s — **다리가 지면 속도를 못 따라가** 미끄러져 보임.
- **해결:** 걷기 17~25px/s, 달리기는 0.38s 주기에 42~54px/s로 **속도-걸음 동기화**.
- **교훈:** 애니메이션 자연스러움은 개별 모션이 아니라 **모션 간 비율**에서 나온다(가이드의 "바운스 주기 = 걸음의 1/2"과 같은 원리).

**4. python이 실행되지 않았는데 "성공"으로 보임**
- **증상:** 카탈로그 수술 스크립트를 python heredoc으로 실행 → 출력 "Python" 한 줄 → tsc 통과 → 그런데 파일 무변경.
- **진짜 원인:** Windows의 **Microsoft Store python 스텁**이 아무것도 안 하고 종료. tsc는 "변경이 없어서" 통과한 것.
- **해결/교훈:** node로 전환. **스크립트 실행 후엔 반드시 결과물을 grep으로 실측**(성공 출력·tsc 통과만으론 부족).

**5. SVG id 충돌 — 렌더 모드 3종으로 해소**
- **문제:** clipPath id가 페이지 내 복제 렌더에서 충돌(가이드의 1번 함정).
- **해결:** 전신=uid 접미사 / 흉상=clip 불필요 레이어만 / **팝업=flat 렌더(clipPath 자체를 안 씀, id 제로)** — 호버 팝업이 페이지에 수십 개 복제돼도 안전.

**6. useState는 prop이 바뀌어도 안 바뀐다**
- **증상:** 새 글 화면에서 다른 폴더 ＋글 클릭 → URL만 바뀌고 상위 폴더 셀렉트 그대로.
- **원인:** 같은 라우트 재렌더 시 컴포넌트가 재마운트되지 않아 useState 초기값 유지.
- **해결:** `useEffect(() => setParentId(initialParentId), [initialParentId])` — 작성 중 본문은 보존.
